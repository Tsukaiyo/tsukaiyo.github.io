What is Accessibility?

We aim to work with the Student Access Collictive at TMU to ensure that our game, once created, is able to be played by everyone.

Please use the dropdown menu below to learn more about different types of accessibility features!

Motor Accessibility Features!

Motor Accessibility Features are game features that are designed to help people with reduced mobility or dexterity enjoy the game experience!

Some examples of these features are:

  • Allowing for controls to be remapped
  • Ensuring the game is compatible with an Adaptive Controller
  • Allowing for control sensitivity to be adjusted
  • Provide alternatives for repeated inputs (button-mashing, quick-time) or held inputs
  • Provide alternatives to timing-based mechanics

Cognitive Accessibility Features!

Cognitive Accessibility Features are game features that are designed to help people with memory, attention, processing, and much more.

Some examples of these features are:

  • Using simple, clear language in an easily readable font size
  • Allowing players to progress through text at their own pace
  • Allow for reminders of controls or objectives during gameplay
  • Provide summaries of progress
  • Give clear indication of what elements can be interacted with

Hearing Accessibility Features!

Hearing Accessibility Features are game features that are designed to help people who are hard of hearing or d/Deaf enjoy playing different games!

Some examples of these features are:

  • Provide subtitles for all speech
  • Provide separate volume controls for effects, music, and speech
  • Provide captions for significant background sounds
  • Provide a visual indication of who is currently speaking
  • Allow for voice chat as an option in multiplayer settings

Vision Accessibility Features!

Vision Accessibility Features are game features that are designed to help people with low vision, colourblindness, or blindness play games!

Some examples of these features are:

  • Provide High Contrast between text/UI and background
  • Use surround sound or screenreader support
  • Provide voiceovers for all text
  • Ensure no essential information is conveyed by colour alone
  • Provide a choice of different designs for cursors, crosshairs, and other important on-screen elements

Speech Accessibility Features!

Speech Accessibility Features are game features that are designed to help people who struggle with verbal communication play different games!

Some examples of these features are:

  • Making sure speech input is not a mandatory feature in your game, or can be turned off
  • For online multiplayer games, allow a preference to be set for players who are willing to play without voice chat
  • Provide visual communication systems for multiplayer games
  • Base speech recognition on volume threshold instead of words
